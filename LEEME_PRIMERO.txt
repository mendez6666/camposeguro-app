CAMPOSEGURO v4.1 — ESCALABLE

Esta versión corrige la base para pensar en cientos de clientes:

1. Multiusuario real.
   Ya no se usa CLIENT_USER_ID por cliente. Cada usuario tiene correo, contraseña/token y sus propias zonas.

2. PostgreSQL obligatorio.
   No subas camposeguro.db a GitHub. La base debe vivir en Render Postgres, Supabase, Neon u otro PostgreSQL.

3. Alertas agrupadas por zona.
   Si una zona tiene muchos focos dentro del radio, CampoSeguro cuenta una zona con alerta, con foco_count y distancia mínima.

4. Correo anti-saturación.
   El sistema prepara máximo un resumen diario por destinatario y urgencias críticas con enfriamiento configurable.

5. Monitor automático.
   En piloto puede correr dentro del web service. En producción conviene usar un background worker separado: python auto_monitor.py
